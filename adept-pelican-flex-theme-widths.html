Adept Pelican Flex Theme widths

Posted on Tue 14 June 2022 in misc

All began with the last article where I posted a simple git command that was rather long and it wasn't displayed fully on my wide screen. This happend to relate to this issue https://github.com/alexandrevicenzi/Flex/issues/196 and the resulting commit https://github.com/alexandrevicenzi/Flex/commit/69fc4c39c10f021d6376c32a50ac777a54e03663 where a max-width: 760px; was added. In https://github.com/alexandrevicenzi/Flex/issues/274#issuecomment-886273776 alexandrevicenzi suggested a workaround using custom CSS to change the value. Fast forward, the workaround worked but the code still had a horizontal scroll bar. Thus I changed the width of the aside container (the left content) and moved the main content to the left. This resulted in the following custom.css data

@media screen and (min-width: 768px) {
        main article {
                max-width: 100%;
        }
        main {
                padding-left: 15vw;
        }
        aside {
                width: 15vw;
        }
}

The custom.css was placed inside my content/extra direcotry. I added the following content to my pelicanconf.conf

STATIC_PATHS = ["extra"]

EXTRA_PATH_METADATA = {
    "extra/custom.css": {"path": "static/css/custom.css"},
}

CUSTOM_CSS = "static/css/custom.css"

Now the full page has a smaller aside container and the main container uses the full screen width.
