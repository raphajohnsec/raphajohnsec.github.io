Clone from Github using SSH and a corporate proxy

Posted on Tue 14 June 2022 in misc

So this is my very first useful post on this blog. You might already know this blog uses Pelican as base and is hosted on Github. To modify and add new content to the page I can either edit thing an Github itself and commit directly. But when I want to edit multiple files it's easier to edit all needed files and commit those and at the end push to Github. Github removed the password authentication, now stating

"Support for password authentication was removed on August 13, 2021. Please use a personal access token instead.".

To clone my source repository I use the SSH option with a private key instead. But how to specify which key the git clone command should use?

Following the reply from here https://stackoverflow.com/questions/6688655/select-private-key-to-use-with-git, I specified the core.sshCommand option. This might have worked when I am not behind a corporate Proxy...

But with this option to specify own SSH commands, we can also add ProxyCommand. This leads to the following command to clone the Github repo.

git clone git@github.com:<REPO_USER>/<REPO_NAME>.git --config core.sshCommand="ssh -i <~/.ssh/PRIVATE_KEY> -o 'ProxyCommand=corkscrew <PROXY_IP> <PROXY_PORT> %h %p'"
